# Bridgeport 1.0.4

Supersedes v1.0.3, which was tagged before the final review pass completed. This release contains everything in v1.0.3 plus two session-semantics fixes found while re-checking that work; use this tag instead of v1.0.3.

## Fixes since v1.0.3

- The legacy `POST /<connector>/message` endpoint now verifies the `sessionId` belongs to the connector in the route, matching the scoping already applied to the Streamable HTTP handlers.
- Streamable HTTP requests that present an `Mcp-Session-Id` for a session that no longer exists (for example, one closed by the idle reaper) now return 404 so the client re-initializes, instead of silently creating a fresh, uninitialized connector process.

## Carried over from v1.0.3

- SSE and Streamable HTTP responses stream chunked byte buffers instead of one byte at a time.
- An idle reaper closes sessions with no open streams and no traffic for 10 minutes, so disconnected clients no longer leak running connector subprocesses.
- `Mcp-Session-Id` reuse, including `DELETE`, is scoped to the connector that issued the session.
- Connector stdin writes moved onto a serial queue; shutdown closes stdin, then escalates SIGTERM to SIGKILL after 5 seconds; connector stdout lines are capped at 32 MB.
- `runShell` drains stderr concurrently, removing a pipe-buffer deadlock in `launchctl` and `cloudflared` invocations.
- The `cloudflared` tunnel lookup no longer misreads Go zero timestamps in `deleted_at` as deletions.
- OAuth access tokens persist privately (`0600`) across daemon restarts, so connected Claude custom connectors survive settings saves without re-authorizing; the client registry is capped at 256 entries.
- GitHub releases publish automatically from `v*` tags or Release workflow dispatches.

## Verification

- All core sources typecheck under Swift 6 strict concurrency and all 38 unit tests pass.
- Live end-to-end smoke test against the real `SSEServer` and mock connector: 401 challenge, legacy SSE round trip, 200KB Streamable HTTP round trip, session reuse, `DELETE` plus stale-session 404 on both DELETE and POST, and `/status`.
- `test_client.py` covers session DELETE and stale-session 404 semantics (12 smoke tests).

This is a source release published from CI. Run `swift test`, `python3 test_client.py`, and `script/build_and_run.sh --verify` on the Mac, then package and notarize the DMG with `script/package_release.sh 1.0.4` and `script/notarize_release.sh`, and attach it to this release.
