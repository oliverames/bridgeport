# Bridgeport 1.0.3

Reliability and robustness release for the Bridgeport local MCP gateway, from a comprehensive review of the daemon serving path.

## Reliability

- SSE and Streamable HTTP responses now stream chunked byte buffers instead of one byte at a time, cutting per-message async overhead by orders of magnitude for large tool responses.
- Sessions track activity and an idle reaper closes sessions with no open streams and no traffic for 10 minutes, so disconnected clients no longer leak running connector subprocesses. Reaped session ids return 404 and clients re-initialize per the Streamable HTTP spec.
- `Mcp-Session-Id` reuse, including `DELETE`, is scoped to the connector that issued the session, so a session for one connector cannot be replayed against another connector's route.
- Connector stdin writes moved onto a serial queue so a full pipe cannot block Swift's cooperative executor; connector shutdown closes stdin, then escalates SIGTERM to SIGKILL after a 5-second grace period; buffered connector stdout lines are capped at 32 MB.
- `runShell` drains stderr concurrently with stdout, removing a pipe-buffer deadlock in `launchctl` and `cloudflared` invocations.
- The `cloudflared` tunnel lookup no longer misreads Go zero timestamps in `deleted_at` as deletions, so **Create or Repair Tunnel** reliably reuses an existing named tunnel.

## OAuth durability

- Issued OAuth access tokens persist privately (`0600`) across daemon restarts, so connected Claude custom connectors survive settings saves without re-authorizing.
- The persisted OAuth client registry is capped at 256 entries, pruning oldest first.

## Release automation

- Pushing an annotated `v*` tag, or dispatching the Release workflow with a version, now publishes a GitHub release automatically via GitHub Actions.

## Verification

- All core sources typecheck under Swift 6 strict concurrency and all 38 unit tests pass (5 new tests: OAuth token persistence and permissions, client registry pruning, session idle detection, chunked stream ordering).
- Live end-to-end smoke test against the real `SSEServer` and mock connector: 401 challenge, legacy SSE round trip, 200KB Streamable HTTP round trip, session reuse, `DELETE` plus stale-session 404, and `/status`.
- `test_client.py` gained a session DELETE smoke test (12 total).

This is a source release published from CI. Run `swift test`, `python3 test_client.py`, and `script/build_and_run.sh --verify` on the Mac, then package and notarize the DMG with `script/package_release.sh 1.0.3` and `script/notarize_release.sh`, and attach it to this release.
