Bridgeport 1.0.5 is a speed, reliability, security, and UX pass over the whole app, focused on making web custom-connector setup (Claude, ChatGPT/Codex, Mistral) dead simple.

## Setup experience

- New **Step-by-Step Setup** guide on every public connector card in the Cloud Connectors pane: numbered, provider-specific instructions with copy buttons for exactly the values each provider asks for (MCP URL and Bridgeport token for Claude, MCP URL for ChatGPT/Codex, MCP URL / `Bearer` header / Mistral JSON payload for Mistral).
- Every copy button now confirms with a transient "Copied" state.
- The query-token fallback is a proper two-way toggle in the Cloud Connectors pane (mirroring Security) instead of a one-way enable button.
- HIG polish: true ellipsis characters in menu and button titles, "Choose…" file buttons with descriptive open-panel messages and prompts, consistent minimum window size, and the non-secret 1Password `op://` reference field is no longer a password field.

## Icon system

- Connector icons now follow the same exposure rules as MCP routes: enabled connectors serve icons locally, and the public hostname serves icons only for Public-toggled connectors. Private connectors advertise a localhost icon URL in `initialize` responses instead of a public URL that would 404.
- The `/icons/<connector>` endpoint answers conditional requests with `ETag`/`304 Not Modified`, so provider caches (Mistral connector cards, Claude connector lists) refresh when artwork changes.
- Icon discovery also checks `logo.png`/`logo.svg` and repo-root `icon.*`/`logo.*` filenames, and `HEAD` responses no longer read the whole file.

## Speed and reliability

- Connector discovery is cached for two seconds on the daemon's serving path, removing a full filesystem walk from every HTTP request.
- MCP messages skip the JSON re-serialization pass unless they actually carry `serverInfo` (icon decoration now costs nothing on regular traffic).
- Live sessions are capped at 64; further session-opening requests get `503` with `Retry-After` so a reconnect-looping client cannot exhaust the Mac with connector subprocesses.
- All `launchctl` interactions in the menu bar app (status checks, daemon install/restart, which retry with sleeps) moved off the main thread, removing multi-second UI freezes during daemon restarts.

## Security

- OAuth token and dynamic-client-registration responses are marked `Cache-Control: no-store` / `Pragma: no-cache` per RFC 6749.
- Failed Bridgeport-token attempts on the OAuth approval page are delayed by one second to slow online guessing.
- Streamable HTTP responses expose `Mcp-Session-Id` via `Access-Control-Expose-Headers` for browser-based MCP clients.

## Verification
