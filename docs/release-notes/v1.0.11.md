# Bridgeport 1.0.11

Bridgeport 1.0.11 keeps cloud MCP connectors signed in after their 12-hour access token expires.

## OAuth refresh tokens

- OAuth clients now receive a refresh token with each access token. Bridgeport rotates the refresh token every time it is used.
- Refresh tokens are bound to the client and MCP resource that received them. A token cannot be reused after rotation.
- Existing token files remain readable. Connections created before this release need one new sign-in because their saved records do not contain a refresh token.

The automated HTTP test now covers the complete PKCE authorization flow, refresh-token rotation, and rejection of a reused refresh token.
